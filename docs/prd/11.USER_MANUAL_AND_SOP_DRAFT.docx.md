## 1. Tentang Dokumen Ini

Dokumen ini adalah draft awal manual pengguna dan SOP untuk setiap role dalam sistem Satu Sehat Kobar. Versi final akan disempurnakan setelah UAT dan pelatihan pilot berdasarkan masukan pengguna nyata.

## 2. Akses Sistem

### 2.1 URL Sistem

Sistem Satu Sehat Kobar dapat diakses melalui browser di alamat:

```
https://ssk.dinkes-kobar.go.id
```

Gunakan browser modern: Google Chrome, Mozilla Firefox, atau Microsoft Edge versi terbaru. Sistem dioptimalkan untuk desktop; akses mobile didukung dengan tampilan responsif.

### 2.2 Login

1. Buka URL sistem di browser
2. Masukkan **Email** (email institusi yang didaftarkan Admin)
3. Masukkan **Password** (diberikan oleh Admin saat akun dibuat)
4. Klik **Masuk**
5. Jika login pertama kali, sistem akan meminta ganti password

### 2.3 Dashboard Setelah Login

Setelah login, pengguna langsung melihat dashboard yang disesuaikan dengan rolenya:

| Role | Tampilan Dashboard |
|------|--------------------|
| Pegawai / Pelaksana | Agenda terbaru, ST saya, jurnal saya |
| Atasan Langsung | Antrian approval, aktivitas tim |
| Kabid / Sekretaris / Kadis | Antrian approval, ringkasan kegiatan unit |
| Keuangan | SPPD menunggu validasi, rekap anggaran |
| Operator Surat | ST yang sudah disetujui, antrian PDF |
| Admin OPD / Admin Faskes | Manajemen user, template, penomoran |
| Auditor | Akses audit trail, tidak ada aksi |
| Reviewer MMC | Draft publikasi menunggu review |

### 2.4 Logout

Klik foto profil atau nama pengguna di pojok kanan atas, lalu pilih **Keluar**.

### 2.5 Reset Password

Jika lupa password:
1. Klik **Lupa Password** di halaman login
2. Masukkan email yang terdaftar
3. Cek email untuk link reset password (berlaku 1 jam)
4. Jika email tidak diterima, hubungi Admin SIK melalui helpdesk

## 3. SOP Pegawai / Pemohon

### 3.1 Membuat Agenda Kegiatan

Agenda adalah pencatatan rencana kegiatan Dinkes sebelum surat tugas dibuat.

**Langkah-langkah:**
1. Dari dashboard, klik menu **Agenda** di navigasi kiri
2. Klik tombol **Buat Agenda Baru**
3. Isi form agenda:
   - **Judul Kegiatan:** nama kegiatan (contoh: "Rapat Koordinasi SPM Gizi")
   - **Tanggal dan Waktu:** tanggal mulai dan selesai
   - **Lokasi:** nama tempat dan alamat
   - **Program/SPM:** pilih program terkait dari daftar
   - **Deskripsi:** uraian singkat tujuan kegiatan
   - **Peserta:** tambah peserta dari daftar pegawai
   - **Lampiran:** upload dokumen undangan atau TOR jika ada
4. Klik **Simpan Agenda**
5. Agenda tersimpan dengan status **Draft** dan dapat diedit sebelum digunakan untuk membuat ST

### 3.2 Membuat ST dari Agenda

Cara termudah dan dianjurkan untuk membuat Surat Tugas:

1. Buka menu **Agenda**, cari agenda yang sudah dibuat
2. Klik **Buat ST dari Agenda** pada baris agenda yang dipilih
3. Sistem otomatis mengisi data ST (judul, tanggal, lokasi, peserta) dari agenda
4. Periksa dan lengkapi:
   - **Jenis ST:** Dalam Kota / Luar Kota / Luar Provinsi
   - **SPPD diperlukan?** Centang jika ada biaya perjalanan
   - Jika SPPD: isi rincian biaya (transportasi, akomodasi, uang harian)
   - **Sumber anggaran:** isi kode anggaran jika berbiaya
5. Klik **Ajukan ST**
6. ST masuk ke antrian approval Atasan Langsung

### 3.3 Membuat ST Manual

Digunakan jika tidak ada agenda yang dibuat terlebih dahulu:

1. Klik menu **Surat Tugas** di navigasi kiri
2. Klik **Buat ST Baru (Manual)**
3. Isi seluruh form secara manual:
   - Judul kegiatan, tanggal, lokasi
   - Peserta (pilih dari daftar pegawai atau tambah manual)
   - Dasar hukum (otomatis dari template)
   - SPPD jika diperlukan
4. Klik **Ajukan ST**

### 3.4 Mengisi Rincian SPPD

SPPD hanya muncul jika ST berjenis "berbiaya" (ada pengeluaran perjalanan dinas):

1. Di form ST, aktifkan toggle **Perlu SPPD**
2. Isi rincian untuk setiap peserta:
   - Moda transportasi (pesawat, darat, laut)
   - Biaya transportasi (pergi + pulang)
   - Jumlah hari
   - Uang harian (otomatis dihitung dari tabel standar)
   - Biaya akomodasi jika menginap
3. Sistem menghitung total otomatis
4. Jika `is_budgeted = false` (tidak ada anggaran formal), centang **Tidak Berbiaya** — langkah validasi keuangan akan di-skip otomatis dalam proses approval

### 3.5 Mengunggah Laporan dan Bukti Tugas

Dilakukan setelah kegiatan selesai dilaksanakan. Semua peserta ST dapat mengunggah bukti:

1. Buka menu **Surat Tugas**, cari ST yang sudah dilaksanakan
2. Klik **Unggah Bukti**
3. Upload dokumen bukti:
   - **Laporan perjalanan:** dokumen laporan kegiatan (PDF/Word)
   - **Bukti transportasi:** foto tiket, struk BBM, boarding pass
   - **Dokumentasi kegiatan:** foto kegiatan, daftar hadir
   - **Bukti biaya lain:** nota hotel, nota makan jika relevan
4. Klik **Simpan Bukti**
5. Atasan Langsung atau Admin Faskes akan memverifikasi bukti

### 3.6 Melihat Jurnal Tugas

Jurnal tugas terbentuk otomatis setelah bukti diverifikasi:

1. Klik menu **Jurnal Saya** di navigasi
2. Lihat daftar kegiatan yang sudah terdokumentasi
3. Klik detail untuk melihat: agenda, ST, bukti, dan status verifikasi
4. Status jurnal: **Draft → Diajukan → Diverifikasi → Selesai**

## 4. SOP Atasan Langsung

### 4.1 Menyetujui Pengajuan ST

1. Setelah login, dashboard menampilkan **Antrian Approval**
2. Klik ST yang menunggu persetujuan
3. Review detail: tujuan kegiatan, peserta, tanggal, rincian SPPD (jika berbiaya)
4. Jika setuju: klik **Setujui** dan opsional tambahkan catatan
5. ST otomatis diteruskan ke langkah approval berikutnya (Kabid atau Sekretaris)

### 4.2 Mengembalikan Pengajuan untuk Revisi

1. Buka ST di antrian approval
2. Klik **Kembalikan untuk Revisi**
3. Wajib isi **Alasan Pengembalian** dengan jelas (contoh: "Tanggal tidak sesuai jadwal rapat rutin, mohon diubah ke tanggal X")
4. Klik **Kirim**
5. Pemohon menerima notifikasi dan dapat merevisi ST
6. Setelah direvisi, ST kembali ke antrian Atasan Langsung

### 4.3 Melihat Dashboard Tim

1. Klik menu **Dashboard Tim** atau **Laporan**
2. Lihat ringkasan: jumlah ST aktif, ST pending approval, kegiatan bulan ini
3. Filter berdasarkan rentang tanggal atau jenis kegiatan

## 5. SOP Kabid / Sekretaris / Kadis (Approver Senior)

### 5.1 Melihat Antrian Persetujuan

1. Dashboard menampilkan tab **Menunggu Persetujuan Saya**
2. Antrian hanya menampilkan ST yang sudah melewati langkah sebelumnya (Atasan Langsung sudah setuju)
3. Jumlah antrian dan detail tanggal pengajuan terlihat di daftar

### 5.2 Proses Approval dengan Catatan

1. Klik ST di antrian
2. Review seluruh detail termasuk catatan Atasan Langsung sebelumnya
3. Klik **Setujui** — opsional tambahkan catatan persetujuan
4. Catatan tercatat di audit trail dan dapat dilihat semua pihak terkait

### 5.3 Penolakan ST dengan Alasan

Penolakan berarti ST tidak dapat dilanjutkan (berbeda dengan pengembalian untuk revisi):

1. Klik **Tolak**
2. Pilih kategori penolakan: "Di luar program prioritas", "Anggaran tidak mencukupi", "Duplikasi kegiatan", atau tulis manual
3. Isi **Alasan Penolakan** secara lengkap
4. Klik **Konfirmasi Penolakan**
5. ST berstatus **Ditolak** dan pemohon menerima notifikasi
6. ST yang ditolak tidak dapat dilanjutkan; pemohon harus membuat ST baru jika perlu

## 6. SOP Keuangan

### 6.1 Validasi Rincian SPPD

Langkah keuangan hanya muncul jika ST memiliki `is_budgeted = true`. ST tanpa anggaran formal di-skip otomatis:

1. Dashboard Keuangan menampilkan **SPPD Menunggu Validasi**
2. Klik SPPD yang akan divalidasi
3. Review rincian:
   - Kode anggaran yang digunakan
   - Total biaya dan rincian per komponen
   - Kesesuaian dengan standar biaya perjalanan dinas
   - Ketersediaan anggaran (cek manual di SIPD atau DPA)
4. Jika valid: klik **Validasi dan Setujui**
5. Jika ada koreksi: klik **Kembalikan dengan Catatan Anggaran**

### 6.2 Mengembalikan dengan Catatan Anggaran

1. Klik **Kembalikan dengan Catatan**
2. Pilih kategori: "Kode anggaran salah", "Komponen biaya tidak sesuai standar", "Anggaran tidak tersedia", atau tulis manual
3. Isi catatan spesifik (contoh: "Gunakan kode MAK 524119, bukan 524113")
4. Klik **Kirim**
5. ST kembali ke pemohon untuk perbaikan rincian SPPD

## 7. SOP Operator Surat

### 7.1 Generate PDF ST

Dilakukan setelah ST melewati seluruh langkah approval:

1. Buka menu **Operator Surat** atau **Antrian PDF**
2. Cari ST dengan status **Siap Cetak**
3. Klik **Generate PDF ST**
4. Sistem menggunakan template dan data ST untuk membuat PDF
5. Periksa preview PDF:
   - Nomor surat (otomatis dari konfigurasi penomoran)
   - Nama dan jabatan pejabat penanda tangan
   - Daftar peserta
   - Tanggal dan lokasi
6. Jika preview sesuai: klik **Konfirmasi dan Unduh PDF**
7. Jika ada kesalahan template: laporkan ke Admin OPD

### 7.2 Generate PDF SPPD

Dilakukan bersamaan atau setelah PDF ST:

1. Dari ST yang sama, klik **Generate PDF SPPD**
2. Sistem membuat PDF SPPD dengan data rincian biaya
3. Periksa: nama peserta, rincian biaya, total, kode anggaran
4. Unduh PDF SPPD

### 7.3 Mengunggah Dokumen Final Bertanda Tangan

Setelah PDF dicetak, ditandatangani secara manual (atau TTE di Phase 2):

1. Scan atau foto dokumen yang sudah bertanda tangan
2. Buka ST di sistem, klik **Unggah Dokumen Final**
3. Pilih file (format: PDF, JPG, PNG; maksimal 10MB)
4. Sistem menghitung hash file untuk verifikasi integritas
5. Klik **Simpan sebagai Dokumen Final**
6. Dokumen tersimpan di arsip digital (immutable — tidak dapat diedit atau dihapus)
7. Pemohon dan peserta dapat mengunduh dokumen final melalui akun mereka

## 8. SOP Admin OPD / Admin Faskes

### 8.1 Kelola Master Data Pengguna dan Role

**Menambah user baru:**
1. Buka **Pengaturan > Manajemen Pengguna > Tambah User**
2. Isi: nama lengkap, email, unit kerja, NIP (opsional)
3. Pilih role yang sesuai fungsi (tidak lebih dari satu role utama)
4. Klik **Simpan** — sistem mengirim link aktivasi ke email user

**Mengubah role:**
1. Cari user di daftar pengguna
2. Klik **Edit** pada baris user
3. Ubah role sesuai perubahan tugas
4. Klik **Simpan** — perubahan tercatat di audit log

**Menonaktifkan user:**
1. Klik **Edit** pada baris user
2. Ubah status ke **Nonaktif**
3. Konfirmasi — user tidak dapat login, data tetap tersimpan

### 8.2 Kelola Template Dokumen

1. Buka **Pengaturan > Template Dokumen**
2. Pilih template yang akan diedit: Template ST, Template SPPD, Template Laporan
3. Gunakan editor template untuk mengubah header, footer, tata letak
4. Klik **Simpan Template**
5. Perubahan berlaku untuk PDF yang di-generate setelahnya

### 8.3 Konfigurasi Penomoran Surat

1. Buka **Pengaturan > Penomoran Surat**
2. Atur format nomor surat: `[Kode Surat]/[Bulan]/[Tahun]`
3. Atur nomor urut terakhir (untuk sinkronisasi dengan nomor surat fisik)
4. Klik **Simpan Konfigurasi**

## 9. SOP Auditor

### 9.1 Mengakses Audit Trail

Akun Auditor bersifat **read-only** — tidak dapat membuat, mengubah, atau menghapus data apapun:

1. Login dengan akun Auditor
2. Buka menu **Audit Trail**
3. Filter log berdasarkan:
   - **Pengguna:** cari aksi dari user tertentu
   - **Jenis aksi:** login, buat ST, approve, generate PDF, upload, dll.
   - **Rentang waktu:** pilih periode audit
   - **Plugin:** filter per modul sistem
4. Setiap baris menampilkan: waktu, user, aksi, data sebelum, data sesudah, IP address

### 9.2 Mengunduh Laporan Audit

1. Setelah filter diterapkan, klik **Unduh Laporan**
2. Pilih format: CSV (untuk analisis) atau PDF (untuk dokumentasi formal)
3. File audit diunduh dengan watermark "RAHASIA — UNTUK KEPERLUAN AUDIT"
4. Simpan dengan aman, jangan disebarluaskan

## 10. Panduan Troubleshooting Umum

| Masalah | Kemungkinan Penyebab | Solusi |
|---------|---------------------|--------|
| Tidak bisa login — "Email atau password salah" | Password salah atau belum diaktifkan | Klik "Lupa Password" atau hubungi Admin SIK |
| Tidak bisa login — "Akun tidak aktif" | Admin telah menonaktifkan akun | Hubungi Admin OPD atau Admin SIK |
| Menu atau tombol tidak muncul | Role tidak memiliki permission untuk fitur ini | Konfirmasi dengan Admin OPD apakah role sudah benar |
| ST tidak muncul di dashboard approval | ST belum sampai di langkah Anda, atau Anda bukan approver di langkah tersebut | Cek status ST di menu "Lacak ST" |
| PDF tidak bisa di-generate — "Template tidak ditemukan" | Template dokumen belum dikonfigurasi | Hubungi Admin OPD untuk setup template |
| PDF tidak bisa di-generate — error umum | Masalah teknis sementara | Tunggu 5 menit dan coba lagi; jika masih gagal laporkan ke helpdesk |
| Dokumen final tidak bisa diunggah — "Format tidak didukung" | File bukan PDF, JPG, atau PNG | Konversi file ke format yang didukung |
| Dokumen final tidak bisa diunggah — "Ukuran file terlalu besar" | File melebihi 10MB | Kompres file atau scan dengan resolusi lebih rendah |
| Bukti tidak bisa diunggah | ST belum berstatus "Sedang Dilaksanakan" atau sudah lewat batas waktu upload | Cek status ST, hubungi Admin jika perlu perpanjangan |
| Dashboard tidak menampilkan data terbaru | KV cache belum diperbarui (cache 15 menit) | Tunggu maksimal 15 menit atau muat ulang halaman |
| Angka di dashboard berbeda dengan ekspektasi | Data belum masuk atau filter periode tidak sesuai | Cek filter tanggal, pastikan data ST sudah diverifikasi |
| "Sesi berakhir, silakan login kembali" | Token autentikasi kedaluwarsa | Login ulang; data yang belum disimpan mungkin hilang |
