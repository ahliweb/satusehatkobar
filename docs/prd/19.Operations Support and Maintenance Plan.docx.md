## 3\. Prinsip Operasional

Operasional sistem mengikuti prinsip:

1. **Stability first**  
   Stabilitas sistem lebih penting daripada penambahan fitur baru saat pilot.

2. **Support accountable**  
   Setiap kendala harus dicatat, diberi status, dan ditindaklanjuti.

3. **Backup before change**  
   Backup wajib sebelum migration, deployment, atau perubahan penting.

4. **Rollback ready**  
   Setiap release harus memiliki rencana rollback.

5. **Least privilege operation**  
   Admin hanya menggunakan akses sesuai kebutuhan.

6. **Security aware support**  
   Support tidak boleh meminta password user, membuka dokumen tanpa hak, atau membagikan data sensitif.

7. **Audit important actions**  
   Aksi penting oleh admin dan user harus tercatat.

8. **Documentation always updated**  
   Perubahan fitur, SOP, dan konfigurasi harus memperbarui dokumentasi.

9. **Issue-driven maintenance**  
   Perbaikan teknis dikelola melalui issue, bukan perubahan acak.

10. **Continuous improvement**  
    Hasil support dan monitoring menjadi bahan perbaikan sistem.

## 13.3 Rollback Plan

Rollback dilakukan jika:

1. Sistem tidak dapat diakses.

2. Migration merusak data.

3. Fitur utama gagal.

4. Permission bypass.

5. Dokumen finance terbuka.

6. PDF gagal total.

7. Upload final gagal total.

8. Bug critical tidak dapat ditangani cepat.

Langkah rollback:

1\. Hentikan release baru.  
2\. Amankan log.  
3\. Restore kode versi sebelumnya.  
4\. Restore database bila diperlukan.  
5\. Restore storage bila diperlukan.  
6\. Jalankan smoke test.  
7\. Informasikan status ke Product Owner.  
8\. Catat incident/deployment report.  
9\. Buat corrective action.
