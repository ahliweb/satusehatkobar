## 2\. Prinsip Monitoring dan Evaluasi

Monitoring dan evaluasi MVP mengikuti prinsip:

1. **Berbasis data.**  
   Penilaian menggunakan data aktual dari sistem, bukan hanya persepsi.

2. **Terukur.**  
   Setiap indikator memiliki definisi, sumber data, dan target.

3. **Bertahap.**  
   Target awal pilot tidak harus sempurna, tetapi harus realistis dan meningkat bertahap.

4. **Berorientasi perbaikan.**  
   Hasil monitoring digunakan untuk memperbaiki sistem, SOP, pelatihan, dan support.

5. **Tidak menghukum user pada masa pilot.**  
   Pilot digunakan untuk pembelajaran dan penyempurnaan.

6. **Role-based.**  
   Evaluasi memperhatikan peran pengguna: pemohon, approver, keuangan, operator surat, pelaksana, verifikator, reviewer MMC, admin, dan pimpinan.

7. **Aman dan privat.**  
   Laporan monitoring tidak boleh membuka data pribadi, data finance, dokumen restricted, atau informasi sensitif tanpa kewenangan.

8. **Actionable.**  
   Setiap indikator yang bermasalah harus memiliki tindak lanjut.

9. **Konsisten.**  
   Definisi indikator tidak berubah-ubah tanpa decision log.

10. **Mendukung keputusan.**  
    Hasil evaluasi digunakan untuk menentukan lanjut pilot, perluasan, perbaikan, atau penundaan.

# 3\. Kerangka Monitoring dan Evaluasi

## 3.1 Level Monitoring

Monitoring dibagi menjadi 5 level:

| Level | Fokus | Contoh |
| :---- | :---- | :---- |
| Input | Kesiapan sumber daya | User terdaftar, role, SOP, training |
| Process | Kelancaran alur kerja | Agenda dibuat, ST disubmit, approval berjalan |
| Output | Hasil langsung | PDF terbit, bukti verified, jurnal terbentuk |
| Outcome | Manfaat operasional | Waktu approval lebih cepat, bukti lebih tertib |
| Impact | Dampak jangka panjang | Tata kelola tugas lebih akuntabel, dashboard pimpinan lebih kuat |

## 21.2 Data Quality Rules

Data KPI harus memenuhi:

1. Ada sumber data jelas.

2. Ada definisi indikator.

3. Ada periode pengukuran.

4. Ada filter unit/faskes bila relevan.

5. Data mengikuti hak akses.

6. Data finance tidak tampil ke user tidak berwenang.

7. Data pribadi tidak ditampilkan berlebihan.

8. Data agregat digunakan untuk dashboard pimpinan.

9. Data anomali dicatat.

10. Perubahan definisi indikator masuk decision log.
