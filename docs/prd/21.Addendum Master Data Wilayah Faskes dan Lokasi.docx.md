# 2\. Plugin Wilayah Resmi Kemendagri

## 2.1 Nama Plugin

awcms-micro-official-region

## 2.2 Tujuan

Plugin ini digunakan untuk mengelola master wilayah resmi berdasarkan kode wilayah administrasi pemerintahan dari Kemendagri.

Wilayah resmi digunakan sebagai referensi utama untuk:

1. Provinsi.

2. Kabupaten/kota.

3. Kecamatan.

4. Desa/kelurahan.

5. Relasi induk wilayah.

6. Validasi wilayah lokal.

7. Faskes.

8. Tujuan ST/SPPD.

9. Dashboard wilayah.

10. Analisis kegiatan per wilayah.

---

## 2.3 Prinsip Plugin Wilayah Resmi

1. Data wilayah resmi tidak diedit bebas oleh user.

2. Perubahan data wilayah dilakukan melalui import/sync resmi.

3. Data memiliki versi sumber.

4. Data memiliki status aktif/nonaktif.

5. Data lama tidak dihapus langsung.

6. Wilayah lokal wajib menginduk ke wilayah resmi.

7. Faskes wajib dikaitkan ke wilayah resmi minimal level desa/kelurahan atau kecamatan sesuai data yang tersedia.

8. ST/SPPD dapat memilih wilayah tujuan dari master resmi.

9. Dashboard dapat melakukan agregasi berdasarkan wilayah resmi.

10. Data wilayah resmi menjadi master referensi, bukan data transaksi.

---

## 2.4 Level Wilayah

Level minimal:

province  
regency\_city  
district  
village\_kelurahan

Istilah field:

province \= provinsi  
regency\_city \= kabupaten/kota  
district \= kecamatan

7. Lokus program.

---

4. Wilayah lokal dapat bertingkat.

5. Wilayah lokal dapat memiliki koordinat.

6. Wilayah lokal dapat digunakan sebagai tujuan kegiatan/ST.

7. Wilayah lokal dapat digunakan untuk dashboard pembinaan.

8. Wilayah lokal dapat dikaitkan dengan faskes binaan.

9. Wilayah lokal memiliki status aktif/nonaktif.

10. Perubahan wilayah lokal tercatat audit.

# 5\. Koordinat Lokasi Agenda dan ST/SPPD

## 5.1 Tujuan

Koordinat lokasi diperlukan agar Agenda dan ST/SPPD memiliki lokasi yang lebih presisi.

Koordinat digunakan untuk:

1. Menentukan titik lokasi kegiatan.

2. Menentukan tujuan ST/SPPD.

3. Mendukung dashboard peta.

4. Mendukung monitoring kunjungan.

5. Mendukung analisis wilayah binaan.

6. Mendukung bukti pelaksanaan tugas.

7. Mendukung pengembangan geospatial fase lanjutan.

---

## 5.2 Penambahan Field pada Agenda

Tambahkan ke agenda\_events:

| Field | Tipe | Keterangan |
| :---- | :---- | :---- |
| official\_region\_id | string/null | Wilayah resmi agenda. |
| local\_region\_id | string/null | Wilayah lokal agenda. |
| health\_facility\_id | string/null | Faskes lokasi/tujuan agenda. |
| location\_name | string/null | Nama lokasi. |
| address\_text | text/null | Alamat manual. |
| latitude | decimal/null | Koordinat lintang. |
| longitude | decimal/null | Koordinat bujur. |
| coordinate\_source | enum/null | manual/faskes/local\_region/gps/import. |
| coordinate\_accuracy\_m | integer/null | Akurasi meter bila tersedia. |

---

## 5.3 Penambahan Field pada ST/SPPD

Tambahkan ke duty\_request\_destinations:

| Field | Tipe | Keterangan |
| :---- | :---- | :---- |
| official\_region\_id | string/null | Wilayah resmi tujuan. |
| local\_region\_id | string/null | Wilayah lokal tujuan. |
| health\_facility\_id | string/null | Faskes tujuan. |
| destination\_name | string | Nama tujuan. |
| address\_text | text/null | Alamat tujuan. |
| latitude | decimal/null | Koordinat lintang. |
| longitude | decimal/null | Koordinat bujur. |
| coordinate\_source | enum/null | manual/faskes/local\_region/gps/import. |
| coordinate\_accuracy\_m | integer/null | Akurasi meter bila tersedia. |
| is\_primary\_destination | boolean | Tujuan utama. |

---

## 5.4 Aturan Koordinat

1. Jika tujuan adalah faskes, koordinat default diambil dari master faskes.

2. Jika tujuan adalah wilayah lokal, koordinat default diambil dari wilayah lokal.

3. Jika tujuan hanya wilayah resmi, koordinat dapat diisi manual.

4. User dapat mengubah koordinat manual bila memiliki permission.

5. Koordinat manual harus memiliki catatan sumber bila diperlukan.

6. Koordinat tidak wajib untuk semua ST pada MVP awal, tetapi sangat direkomendasikan.

7. Koordinat wajib untuk dashboard peta/geospatial fase lanjutan.

8. Koordinat tidak boleh digunakan untuk tracking real-time pegawai pada MVP.

9. Koordinat bukti GPS presensi tidak masuk MVP kecuali diputuskan.

10. Perubahan koordinat penting harus diaudit.

---

## 5.5 Coordinate Source Enum

manual  
health\_facility  
local\_region  
official\_region  
gps  
import  
unknown
