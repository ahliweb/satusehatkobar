## 7\. Strategi Atomic Development

AI coding agent harus bekerja dalam satuan kecil.

### 7.1 Aturan Atomic

1. Satu issue \= satu fokus kerja.

2. Satu commit \= satu perubahan logis.

3. Jangan mencampur database, UI besar, dan refactor besar dalam satu commit tanpa alasan.

4. Migration harus terpisah dan jelas.

5. API harus diuji sebelum UI bergantung penuh.

6. UI harus mengikuti pola komponen yang ada.

7. Dokumentasi harus diperbarui pada issue yang relevan.

8. Test minimal harus ditambahkan atau dijalankan.

9. Jika perubahan terlalu besar, pecah menjadi beberapa issue.

10. Jika ada blocker, hentikan dan laporkan.

### 7.2 Urutan Pekerjaan Atomic

Urutan umum:

1. Analisis pola existing.

2. Buat/update issue.

3. Buat migration.

4. Buat service/handler.

5. Buat API route.

6. Buat permission.

7. Buat UI.

8. Tambahkan audit trail.

9. Tambahkan seed bila diperlukan.

10. Tambahkan test.

11. Update dokumentasi.

12. Jalankan validasi.

13. Commit.

14. Laporkan hasil.

1. Semua endpoint state-changing wajib authentication.

2. Semua endpoint wajib permission check.

3. Endpoint yang membaca dokumen wajib ABAC.

4. Response error harus jelas.

6. Business logic ditempatkan di service/handler sesuai pola repo.
