# IMPLEMENTATION\_BACKLOG.md

## Backlog Implementasi MVP Satu Sehat Kobar v1.4

### Agenda Dinkes, ST/SPPD, Bukti Tugas, SPM, Jurnal Pegawai, Dashboard, MMC Terbatas, dan Arsip Dasar

---

## 1\. Tujuan Dokumen

Dokumen ini adalah backlog implementasi teknis untuk MVP **Satu Sehat Kobar v1.4** berbasis **AWCMS-Micro**.

Dokumen ini digunakan untuk:

1. Membagi pekerjaan menjadi epic dan atomic issue.

2. Menjaga implementasi tetap sesuai scope MVP.

3. Mencegah pekerjaan melebar ke fitur fase lanjutan.

4. Menjadi dasar pembuatan GitHub milestone dan GitHub issues.

5. Menjadi acuan AI coding agent dalam mengerjakan implementasi secara bertahap.

6. Menjadi alat tracking progres Tim SIK.

7. Menjadi dasar validasi sprint, UAT, dan go-live pilot.

---

## 2\. Prinsip Backlog

Backlog ini mengikuti prinsip berikut:

1. **Atomic** — satu issue harus kecil, jelas, dan dapat diuji.

2. **Traceable** — setiap issue harus dapat dilacak ke PRD, MVP, dan acceptance criteria.

3. **Plugin-first** — implementasi dilakukan melalui plugin, template, route, handler, service, migration, permission, docs, dan test.

4. **No core fork** — core AWCMS-Micro/EmDash tidak diubah kecuali benar-benar diperlukan.

5. **Security-aware** — setiap fitur memperhatikan RBAC, ABAC, audit trail, dan proteksi data.

6. **Documentation-first** — perubahan penting harus memperbarui dokumentasi.

7. **Testable** — setiap issue memiliki validasi minimal.

8. **No scope creep** — fitur Won’t Have MVP tidak boleh masuk backlog implementasi MVP.
