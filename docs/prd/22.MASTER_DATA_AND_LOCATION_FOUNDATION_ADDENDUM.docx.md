# MASTER\_DATA\_AND\_LOCATION\_FOUNDATION\_ADDENDUM.md

## Addendum Fondasi Master Data, Wilayah, Faskes, Organisasi, Pegawai, Penandatangan, Penomoran, SPM, dan Lokasi ST

### Tambahan Resmi untuk Paket Dokumen MVP Satu Sehat Kobar v1.4 Berbasis AWCMS-Micro

---

## 1\. Status Dokumen

Dokumen ini merupakan **dokumen tambahan resmi** untuk melengkapi paket dokumen MVP **Satu Sehat Kobar v1.4**.

Dokumen ini ditambahkan karena paket dokumen sebelumnya sudah memuat alur Agenda, ST/SPPD, Approval, PDF, Bukti, Jurnal, Dashboard, MMC, Arsip, Security, UAT, Monitoring, Risiko, Data Governance, Integrasi, dan Operasional, tetapi belum mengunci secara eksplisit beberapa master data fondasi.

Master data fondasi yang ditambahkan:

1. Wilayah resmi sesuai referensi Kemendagri.

2. Wilayah lokal yang menginduk ke wilayah resmi level desa/kelurahan.

3. Faskes sebagai tujuan ST/SPPD.

4. Koordinat lokasi Agenda dan ST/SPPD.

5. Unit organisasi Dinkes/faskes.

6. Pegawai lokal/snapshot pegawai.

7. Pejabat penandatangan.

8. Penomoran naskah dinas/ST/SPPD.

9. Master SPM/indikator/program.

10. Kalender hari libur/cuti bersama.

11. Master transport/alat angkut.

12. Template approval chain.

---

## 2\. Tujuan Dokumen

Dokumen ini bertujuan untuk:

1. Menambahkan fondasi master data yang belum eksplisit.

2. Menjamin Agenda dan ST/SPPD memiliki referensi wilayah yang valid.

3. Menjamin ST/SPPD dapat memilih tujuan faskes secara terstruktur.

4. Menjamin lokasi ST dapat memiliki koordinat.

5. Menjamin dashboard wilayah/faskes dapat dibuat secara valid.

6. Menjamin struktur organisasi, pegawai, pejabat, dan penomoran dokumen lebih tertib.

7. Menjamin SPM/program dapat dikaitkan dengan Agenda, ST, bukti, dan dashboard.

8. Menyiapkan pengembangan fase lanjutan seperti SATUSEHAT, Khanza, GIS, dashboard faskes, dan public portal.

9. Menjaga implementasi tetap berbasis plugin AWCMS-Micro.

10. Menjadi addendum untuk PRD, arsitektur plugin, database schema, API contract, sprint plan, UAT, dashboard KPI, dan master document index.

---

## 3\. Prinsip Umum Master Data

Semua master data dalam dokumen ini mengikuti prinsip:

1. **Single source of reference**  
   Data master digunakan sebagai referensi utama agar input tidak berulang dan tidak liar.

2. **Plugin-first**  
   Master data dibuat sebagai plugin/komponen modular, bukan logika tersebar di fitur Agenda atau ST/SPPD.

3. **Traceable**  
   Perubahan master data penting harus tercatat audit.

4. **Controlled update**  
   Data resmi tidak boleh diubah bebas oleh user biasa.

5. **Snapshot for historical documents**  
   Dokumen ST/SPPD final harus menyimpan snapshot data penting agar riwayat tidak berubah ketika master data berubah.

6. **No clinical data in master**  
   Master faskes dan pegawai tidak memuat data klinis pasien.

7. **ABAC-ready**  
   Master wilayah, organisasi, faskes, dan pegawai harus dapat digunakan untuk pembatasan akses berbasis unit/faskes/wilayah.

8. **Dashboard-ready**  
   Master data harus dapat menjadi basis rekap dashboard wilayah, faskes, SPM, unit, dan kegiatan.

9. **Future integration-ready**  
   Master data disiapkan agar kelak dapat diintegrasikan dengan SIMPEG, SATUSEHAT, Khanza, SRIKANDI, SIPD, dan sistem lain.

10. **Secure by default**  
    Data kontak, pejabat, pegawai, dan faskes tidak boleh dipublikasikan tanpa keputusan.

* Faskes memiliki wilayah resmi.

* Faskes dapat memiliki koordinat.

* Faskes dapat dipilih sebagai tujuan ST.

* Dashboard dapat menampilkan rekap per faskes.

## UAT-018 — Koordinat Lokasi ST

* ST dari faskes mengambil koordinat faskes.

* ST dari wilayah lokal mengambil koordinat wilayah lokal bila tersedia.

* User dapat mengisi koordinat manual bila berwenang.

* Koordinat tersimpan pada tujuan ST.

* Koordinat tampil pada detail ST.

* Koordinat tidak menjadi tracking real-time pegawai.

## UAT-019 — Unit Organisasi

* Admin dapat membuat struktur unit.

* Unit dapat memiliki parent-child.

* User/pegawai dapat dikaitkan ke unit.

* Dashboard dapat difilter berdasarkan unit.

* ABAC unit berjalan.

## UAT-020 — Master Pegawai Lokal

* Admin dapat membuat pegawai.

* Pegawai dapat dikaitkan dengan user.

* Pegawai dapat dipilih sebagai pelaksana.

* Snapshot pegawai tersimpan pada ST.

* Perubahan master pegawai tidak mengubah dokumen final.

## UAT-021 — Pejabat Penandatangan dan Penomoran

## UAT-022 — Master SPM/Program

* Admin dapat membuat master SPM/program.

* Agenda dapat dikaitkan ke SPM/program.

* ST dapat dikaitkan ke SPM/program.

* Dashboard SPM menggunakan master tersebut.
